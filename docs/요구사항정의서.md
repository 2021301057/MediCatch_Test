# MediCatch 요구사항 정의서 v2.0

> 기준: 헥토데이터 요구 기술 스택 (Java/Spring Boot, React, MySQL, GitHub/Jenkins)  
> 아키텍처: MSA (Spring Cloud Gateway + Eureka + Kafka + RabbitMQ)  
> 개발 기간: 2026년 4월 ~ 6월 (3개월)

---

## 1. 시스템 개요

**서비스명**: MediCatch  
**슬로건**: "내 건강 데이터로 찾는, 나에게 딱 맞는 보험"  
**핵심 데이터**: CODEF 헬스케어 패키지 + CODEF 보험사 API  

---

## 2. 기능 요구사항 (전체 7개 화면)

---

### 기능 1. 진료 전 검색 ⭐ 신규

**목적**: 병원 가기 전에 "이 치료 보험 되나?" 미리 확인

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Analysis Service (8004) + Insurance Service (8003) |
| 사용 CODEF API | 보험사 API (coverage_items) |
| 사용 DB 테이블 | `policies`, `coverage_items` |
| Frontend | 검색 입력창 + 치료별 보장 결과 카드 |

**세부 기능**:
- 치료명/시술명 키워드 검색 (ex: "도수치료", "MRI", "백내장")
- 내가 가입한 보험별 보장 여부 + 보장 금액 표시
- 보험 세대(1~4세대 실손)별 보장 조건 차이 안내
- 예상 본인부담금 계산 (총비용 - 보험금)
- 비급여 항목 여부 표시

**API 엔드포인트**:
```
GET /api/analysis/pre-treatment-search?keyword={치료명}
GET /api/analysis/pre-treatment-search/{treatmentId}/coverage
```

### 기능 2. 건강 검진 기록 내역

**목적**: CODEF로 가져온 건강검진 결과를 한눈에 확인

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Health Service (8002) |
| 사용 CODEF API | 건강검진결과 조회, 건강나이알아보기, 검진대상 조회 |
| 사용 DB 테이블 | `checkup_results`, `disease_predictions` |
| Frontend | 연도별 검진 결과 카드 + 수치 차트 |

**세부 기능**:
- 연도별 건강검진 결과 목록 (혈압, 혈당, 콜레스테롤, BMI 등)
- 건강나이 vs 실제나이 비교 시각화
- 미수검 검진 항목 안내 (암검진, 일반검진, 구강검진)
- 뇌졸중·당뇨·심뇌혈관 위험도 예측 결과
- 위험 등급별 색상 표시 (HIGH=빨강, MEDIUM=주황, LOW=초록)

**API 엔드포인트**:
```
GET /api/health/checkup-results
GET /api/health/checkup-results/{year}
GET /api/health/disease-predictions
GET /api/health/checkup-targets  ← 미수검 항목
```

### 기능 3. 내 보험 조회

**목적**: 가입한 모든 보험사 보험을 한 화면에서 통합 조회

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Insurance Service (8003) |
| 사용 CODEF API | CODEF 보험사 API (삼성화재, 한화생명 등) |
| 사용 DB 테이블 | `policies`, `coverage_items`, `policy_premiums` |
| Frontend | 보험사별 카드 UI + 보장 항목 상세 |

**세부 기능**:
- 전체 보험 계약 목록 (생명/손해/실손/암/치아 종류별 분류)
- 보험사명, 상품명, 월납 보험료, 만기일, 계약 상태 표시
- 보장 항목별 보장 금액 상세 조회 (사망, 암, 뇌졸중, 입원비 등)
- 월 총 보험료 합계 + 소득 대비 보험료 비율 경고
- 보장 카테고리별 총 보장 금액 파이차트

**API 엔드포인트**:
```
GET /api/insurance/policies
GET /api/insurance/policies/{policyId}/coverage
GET /api/insurance/summary  ← 월 보험료 합계, 보장 총액
```

### 기능 4. 진료 기록 & 보험 청구

**목적**: 진료 내역 자동 수집 + 청구 가능한 보험금 탐지 & 청구 안내

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Health Service (8002) + Analysis Service (8004) |
| 사용 CODEF API | 진료받은 내용 조회, 진료 및 투약정보 조회 |
| 사용 DB 테이블 | `medical_records`, `medication_details`, `claim_opportunities` |
| Frontend | 진료 내역 리스트 + 청구 가능 알림 배지 |

**세부 기능**:
- CODEF 연동으로 진료 내역 자동 수집 (병원명, 진료일, 본인부담금)
- 투약 내역 조회 (약품명, 투약일수)
- 진료 기록 × 보험 약관 자동 매핑 → 청구 가능 금액 산출
- 청구 기회 알림 (Kafka → RabbitMQ → 실시간 푸시)
- 청구 완료 이력 관리 (중복 청구 방지)
- 보험사별 청구 절차 링크 제공

**API 엔드포인트**:
```
GET /api/health/medical-records
GET /api/health/medical-records/{recordId}/medications
GET /api/analysis/claim-opportunities
POST /api/analysis/claim-opportunities/{claimId}/complete
```

### 기능 5. 보험 추천 & 보장 공백

**목적**: 현재 보험의 빈 구멍을 찾고, 맞춤 보험 상품 추천

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Analysis Service (8004) + Recommend Service (8005) |
| 사용 CODEF API | 보험사 API + 건강보험공단 질환 예측 |
| 사용 DB 테이블 | `coverage_gap_reports`, `insurance_recommendations`, `health_profiles` |
| Frontend | 보장 공백 게이지 차트 + 추천 상품 카드 |

**세부 기능**:
- 건강 위험도(뇌졸중·당뇨·심뇌혈관) × 현재 보장 내역 교차 분석
- 보장 공백 카테고리별 시각화 (암, 뇌질환, 심혈관, 치아 등)
- 위험도 기반 맞춤 보험 상품 추천 (광고 아님, 데이터 기반)
- 보험료 최적화 시뮬레이션 (중복 보장 제거 + 부족 보장 강화)
- 건강나이 활용 보험료 할인 가능 상품 안내

**API 엔드포인트**:
```
GET /api/analysis/coverage-gap
GET /api/recommend/insurance-products
GET /api/recommend/premium-optimization
```

### 기능 6. 건강 통합 리포트

**목적**: 최근 12개월 의료 이용 패턴 + 보험 활용 현황 종합 리포트

| 항목 | 내용 |
|------|------|
| 담당 서비스 | Analysis Service (8004) |
| 사용 CODEF API | 진료 내역 + 건강검진 결과 + 보험사 API |
| 사용 DB 테이블 | `medical_records`, `checkup_results`, `policies`, `health_profiles` |
| Frontend | 월별 Bar Chart + 위험도 Line Chart + 요약 카드 |

**세부 기능**:
- 최근 12개월 월별 의료비 지출 추이 (Bar Chart)
- 급여 vs 비급여 비율 파이차트
- 보험 청구 vs 미청구 금액 비교
- 뇌졸중·당뇨·심뇌혈관 위험도 연도별 변화 추적 (Line Chart)
- 예방접종 현황 + 미접종 항목 알림
- 자주 방문 병원/진료과 통계
- 월별 리포트 PDF 다운로드

**API 엔드포인트**:
```
GET /api/analysis/health-report?period=12months
GET /api/analysis/health-report/medical-patterns
GET /api/analysis/health-report/insurance-usage
```

### 기능 7. 건강 채팅 ⭐ 신규

**목적**: 내 건강·보험 데이터를 바탕으로 AI와 자유롭게 대화

| 항목 | 내용 |
|------|------|
| 담당 서비스 | **신규 Chat Service (8007)** |
| 사용 기술 | OpenAI API (GPT-4o) + RAG + 사용자 컨텍스트 주입 |
| 사용 DB 테이블 | `chat_history` (신규), `health_profiles`, `policies` |
| Frontend | 채팅 UI (말풍선) + 빠른 질문 버튼 |

**세부 기능**:
- 자연어 질문으로 보험 보장 여부 확인 ("도수치료 보험 돼?")
- 내 건강 데이터 기반 맞춤 답변 (건강나이, 위험도 반영)
- 진료 전 체크리스트 생성 ("정형외과 가기 전에 뭐 챙겨야 해?")
- 보험 청구 절차 안내 ("실손 청구 어떻게 해?")
- 대화 이력 저장 + 이전 대화 이어서 보기
- 빠른 질문 버튼 (자주 묻는 질문 5가지 제공)

**신규 DB 테이블 필요**:
```sql
-- Chat Service 스키마 (프로젝트_chat)
CREATE TABLE chat_history (
  chat_id     BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id     BIGINT NOT NULL,
  role        ENUM('USER', 'ASSISTANT'),
  message     TEXT NOT NULL,
  context_json JSON,          -- 답변 생성에 사용된 데이터 컨텍스트
  created_at  DATETIME DEFAULT NOW()
);
```

**API 엔드포인트**:
```
POST /api/chat/message         ← 질문 전송
GET  /api/chat/history         ← 대화 이력 조회
DELETE /api/chat/history       ← 대화 초기화
```

## 4. 프론트엔드 화면 구성

```
[사이드바 메뉴]
├── 🏠 대시보드          ← 보험 요약 + 청구 알림
├── 🔍 진료 전 검색       ← 기능 1 (신규)
├── 🏥 건강 검진 기록     ← 기능 2
├── 🛡️ 내 보험 조회      ← 기능 3
├── 📋 진료 기록 & 청구   ← 기능 4
├── 📊 보험 추천 & 공백   ← 기능 5
├── 📈 건강 통합 리포트   ← 기능 6
└── 💬 건강 채팅          ← 기능 7 (신규)
```

## 5. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답 시간 | API 3초 이내 |
| 보안 | CODEF Connected ID AES-256 암호화 저장 |
| 인증 | JWT Access(15분) + Refresh(7일) |
| 가용성 | Resilience4j 서킷브레이커 (CODEF API 장애 시 캐시 반환) |
| 로그 | 건강 데이터 조회 이력 감사 로그 별도 테이블 기록 |
